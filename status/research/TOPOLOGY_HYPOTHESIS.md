# Topology Hypothesis

Purpose: state the orchestration hypothesis in research language, grounded in the retained Hearthfield evidence.

## Hypothesis

A large autonomous build does not require every agent to share the whole project context.

It requires:

- one high-context control plane
- strong disk state
- bounded foremen
- narrow workers

In this model, the top-level orchestrator is not a universal implementer.
It is a coherence-preserving control plane.

## Strong Form

If the code structure and orchestration structure are both strong enough, then:

- one high-context chat can hold the core plan
- foremen can own bounded surfaces
- workers can remain local and disposable
- drift is gated by structure, not by asking every agent to stay globally aligned

## Observed Support

These points are supported by retained Hearthfield evidence:

- top-level context pressure directly changed architecture decisions
- live repo state and short operational doctrine shaped behavior more than long theory docs
- sessions repeatedly show the value of a single state file and a single control-plane perspective
- resumed session self-reports say the highest-yield boot bundle is operational, not theoretical

## What This Hypothesis Is Not

It is not:

- "one chat does all the work"
- "many agents all share full context"
- "docs alone prevent drift"

It is:

- one chat holds global truth
- bounded lanes hold local truth
- validated state moves upward
- narrow instructions move downward

## Main Predicted Benefits

If the hypothesis is correct, this topology should:

- reduce top-level context burn
- reduce unnecessary worker reading
- improve integration coherence
- keep global plan changes legible
- reduce ghost progress caused by lane-local drift being mistaken for global progress

## Main Failure Modes

The topology should fail when:

- disk state is weak
- foremen are not bounded
- integration is delayed too long
- top-level reads every worker log
- lane state is preserved only in chat

## Most Important Open Question

Can another team adopt the primitive and get most of the effect?

Current evidence says:

- partially, yes, at the level of doctrine and workflow
- not yet proven, at the level of robust transfer across operators

That is why the next research move is not more prose.
It is a transfer experiment with:

- standardized boot bundle
- standardized lane ledger
- standardized scoring
- external operators
